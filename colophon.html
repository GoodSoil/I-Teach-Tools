Colophon How this was built

This provides a summary of how this Angular 2 app was built and the dependencies in this app.

Application Setup

  • Make a directory and run npm init. Add .gitignore (from gitignore.io) and .gitattributes (for Windows).
  • Use npm to install Angular 2 and its key dependencies (zone.js and rxjs). Remember to use the --save option to include those packages in the site's package.json file. One of the things that you will encounter if you use npm to install Angular manually are notices of peer dependencies that Angular 2 has on other packages; peer dependencies are not loaded by default, and you will need to explicitly load all of them for your development. When you are done, your package.json will show your project's fundamental dependencies.
  • Install developer dependencies:
    • TypeScript and Typings via npm (--save-dev)
    • lite-server - for previewing/testing on the local machine
    • concurrently - for running console commands at the same time, such as tsc (TypeScript Compiler) and lite-server. This is especially helpful when such commands are set up to watch for changes to source files.
    • tslint - optional, but helpful for enforcing standards in your coding
  • Configure TypeScript and Typings (tsconfig.json and typings.json)
    • TypeScript. Run tsc --init to create the tsconfig.json file.
    • Typings. You need to run typings install dt~es6-shim --save --global (as per this issue)
  • NPM Scripts in package.json
  • Initial index.html Page using CDNs (with embedded system configuration and bootstrapping)
  • Create Site Layout Component as Root Component
  • Test-run the Application

Core Development

The primary aim of this particular application is to help in creating Learning Outcome Guides for classroom delivery of course materials.

Post-development Packaging

The goals of the post-development packaging is to deliver this app as a Windows application.
